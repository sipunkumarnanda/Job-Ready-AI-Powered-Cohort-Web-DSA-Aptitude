### 1. What are HTTP Headers?

* **HTTP Headers** are **key-value pairs** that are sent along with every **HTTP request** and **HTTP response** between a **client** (like a browser or app) and a **server**.
* They carry **extra information** about the request or response â€” called **metadata**.
* The HTTP protocol uses headers to describe *how* to handle the message, not just *what* the message is.

### 3. Two Main Types of HTTP Headers

| Type                 | Description                                                               | Example                                                 |
| -------------------- | ------------------------------------------------------------------------- | ------------------------------------------------------- |
| **Request Headers**  | Sent **from client to server**. Provide info about the request or client. | `Content-Type`, `Authorization`, `Cookie`, `User-Agent` |
| **Response Headers** | Sent **from server to client**. Provide info about the response.          | `Content-Type`, `Set-Cookie`, `Cache-Control`, `Server` |

---

### 4. Common **Request Headers**

| Header            | Purpose                                                                                        |
| ----------------- | ---------------------------------------------------------------------------------------------- |
| **Host**          | Domain name or IP of the server                                                                |
| **Content-Type**  | Type/format of data being sent (`application/json`, `application/x-www-form-urlencoded`, etc.) |
| **Authorization** | Carries tokens or credentials for authentication                                               |
| **Cookie**        | Sends cookies from the client to the server                                                    |
| **Accept**        | Tells server which response types the client accepts (e.g. JSON, HTML)                         |
| **User-Agent**    | Identifies the browser or app making the request                                               |
| **Referer**       | Shows the previous page that made the request                                                  |

---

### 5. Common **Response Headers**

| Header                          | Purpose                                              |
| ------------------------------- | ---------------------------------------------------- |
| **Content-Type**                | Tells the client what format the response data is in |
| **Set-Cookie**                  | Sends cookies from the server to the browser         |
| **Cache-Control**               | Tells browser whether it can cache this response     |
| **Server**                      | Shows information about the backend server           |
| **Access-Control-Allow-Origin** | Used for CORS (Cross-Origin Resource Sharing)        |
| **Date**                        | The date and time when the response was sent         |

---

### 6. Why Headers Are Important

* They help **control communication** between client and server.
* They define **data type**, **authentication**, **cookies**, and **security policies**.
* Many Express middlewares rely on headers:

  * `express.json()` â†’ uses `Content-Type`
  * `cookieParser()` â†’ uses `Cookie`
  * Authentication â†’ uses `Authorization`
  * CORS â†’ uses `Access-Control-Allow-Origin`

### 7. Accessing Headers in Express.js

#### To get all headers:

```js
app.get('/', (req, res) => {
  console.log(req.headers)
})
```

#### To get a specific header:

```js
req.get('Content-Type')
req.get('Authorization')
req.get('Cookie')
```

#### To set response headers:

```js
res.set('Content-Type', 'application/json')
res.set('Cache-Control', 'no-cache')
```
