

# üü¢ Quick summary ‚Äî what ‚ÄúFree EC2‚Äù means

* New AWS accounts are typically eligible for **750 hours/month** of micro instance usage (enough to run **one** micro instance full time for a month). Use **t2.micro** (or **t3.micro** in regions where t2 is unavailable) to stay inside the classic Free Tier. ([Amazon Web Services, Inc.][1])
* **Always check your account‚Äôs Free Tier offers and track usage** ‚Äî AWS has changed offers based on account creation dates and plan types. Use the Billing/Free Tier dashboard to avoid surprises. ([AWS Documentation][2])

# ‚úÖ Before you start (prerequisites & safety)

* Create an **AWS account** and sign in as an **IAM user** with permissions to create EC2 resources (don‚Äôt use the root account).
* Choose a **region** near you (Free-Tier availability and AMIs vary by region).
* Decide whether you‚Äôll use the **Console** (easier) or the **AWS CLI** (scriptable).
* Make sure you have a secure place to store a downloaded key pair (`.pem`) ‚Äî **don‚Äôt check it into git**.

# 1. Console method ‚Äî step-by-step (recommended for first time users) üñ±Ô∏è

## 1.1 Go to the EC2 Dashboard

1. Sign in to the AWS Management Console.
2. Services ‚Üí **EC2** ‚Üí **Instances** (or ‚ÄúLaunch instances‚Äù). ([Amazon Web Services, Inc.][3])

## 1.2 Click **Launch instances** ‚ûï

Gives you the Launch instance wizard with the following main sections.

### 1) Choose an Amazon Machine Image (AMI) üñºÔ∏è

* Pick an **Amazon Linux 2** AMI or an **Ubuntu Server** AMI that is **Free-Tier eligible**. Amazon recommends Amazon Linux 2 + t2.micro for getting started. ([Amazon Web Services, Inc.][3])

### 2) Choose an Instance Type ‚öôÔ∏è

* Select **t2.micro** (or **t3.micro** where t2 is not available) to stay within Free Tier. Double-check the UI label that says ‚ÄúFree tier eligible‚Äù. ([Amazon Web Services, Inc.][1])

### 3) Configure Instance Details üß≠

* **Network (VPC)**: choose the default VPC unless you have a custom VPC.
* **Subnet**: pick a subnet (public subnet if you want public SSH access).
* **Auto-assign Public IP**: Enable if you want to SSH from the Internet (or use EC2 Instance Connect).
* **IAM role**: optional ‚Äî add a role if your instance needs to call other AWS services.
* **Shutdown behavior**: default is Stop ‚Äî good for preserving volume and avoiding recreation.

> Tip: Don‚Äôt enable unnecessary features (like detailed monitoring) that may incur extra charges.

### 4) Add Storage üíæ

* The default root volume is usually modest (often 8 GiB). **Keep it small** for Free Tier and reduce additional costs. If the launch wizard offers a larger default, shrink it to 8 GiB (or the smallest reasonable size) for a micro instance.

### 5) Add Tags üè∑Ô∏è

* Add a `Name` tag (e.g., `Name = my-free-ec2`) so it‚Äôs easy to find later.

### 6) Configure Security Group (firewall) üîí

* Create a new Security Group. Recommended rules for a simple Linux web/dev instance:

  * **SSH (TCP 22)** ‚Üí **Source:** your IP only (e.g., `203.0.113.45/32`) or use `My IP` button in console.
  * **HTTP (TCP 80)** ‚Üí Source `0.0.0.0/0` only if you need a webserver accessible publicly.
  * **HTTPS (TCP 443)** ‚Üí Source `0.0.0.0/0` if needed.
* **Important:** Don‚Äôt open SSH to `0.0.0.0/0` ‚Äî that‚Äôs unsafe.

### 7) Key Pair ‚Äî create or select üîë

* **Create a new key pair** (RSA or ED25519) and **download the `.pem`** file.
* Save it in a secure place and run `chmod 400 mykey.pem` locally on Linux/macOS before using.
* If you lose the key, you can‚Äôt SSH into the instance (unless you use EC2 Instance Connect or have another user/role set up).

### 8) Review & Launch üöÄ

* Click **Launch instance**. Wait a minute for the instance state to become **running**.

# 2. Connect to your instance (SSH) üîó

## Find the Public IP / DNS

* In the EC2 Console ‚Üí Instances ‚Üí your instance ‚Üí **Public IPv4 address** or **Public DNS**.

## SSH from macOS / Linux / WSL:

```bash
chmod 400 MyKeyPair.pem       # only once (on the file you downloaded)
ssh -i MyKeyPair.pem ec2-user@3.121.45.67
```

* `ec2-user` is the default for Amazon Linux AMIs. For Ubuntu AMIs use `ubuntu@‚Ä¶`, for Debian `admin@‚Ä¶` etc.

## EC2 Instance Connect (Browser method)

* Console ‚Üí Instances ‚Üí Select instance ‚Üí **Connect** ‚Üí **EC2 Instance Connect** (works for Amazon Linux/Ubuntu in supported regions) ‚Äî useful if you lost the key or want quick browser access. ([Amazon Web Services, Inc.][3])

# 3. CLI method ‚Äî create & launch from terminal (advanced / scriptable) üß∞

## 3.1 List free-tier eligible instance types

(This uses AWS CLI; ensure you‚Äôve configured `aws configure` with credentials and the correct region.)

```bash
aws ec2 describe-instance-types \
  --filters Name=free-tier-eligible,Values=true \
  --query "InstanceTypes[*].[InstanceType]" \
  --output text | sort
```

This lists the instance types marked free-tier eligible for your account/region. ([AWS Documentation][4])

## 3.2 List free-tier eligible AMIs (example)

```bash
aws ec2 describe-images \
  --filters Name=free-tier-eligible,Values=true Name=is-public,Values=true \
  --owners amazon \
  --query "Images[*].[ImageId,Name]" --output text | sort
```

Pick an AMI ID from the output (e.g., an Amazon Linux 2 AMI) and note the `ImageId`. ([AWS Documentation][4])

## 3.3 Create a key pair via CLI (example)

```bash
aws ec2 create-key-pair --key-name MyKeyPair --query 'KeyMaterial' --output text > MyKeyPair.pem
chmod 400 MyKeyPair.pem
```

## 3.4 Launch instance with `run-instances`

```bash
aws ec2 run-instances \
  --image-id ami-0abcdef1234567890 \
  --count 1 \
  --instance-type t2.micro \
  --key-name MyKeyPair \
  --security-group-ids sg-0123456789abcdef0 \
  --subnet-id subnet-0abcd1234ef567890 \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=my-free-ec2}]'
```

* Replace `ami-‚Ä¶`, `sg-‚Ä¶`, and `subnet-‚Ä¶` with values from your account.

# 4. Cost & Free-Tier gotchas ‚Äî **watch these** ‚ö†Ô∏è

* **750 hours/month** covers *one* micro instance running full time ‚Äî running multiple micro instances or running the same instance type beyond 750 hours will incur charges. ([Amazon Web Services, Inc.][1])
* **t3.micro default to Unlimited mode** in some cases ‚Äî if you burst CPU frequently in Unlimited mode you can get charged for CPU credits. If you‚Äôre on t3, check the CPU credit mode. ([AWS Documentation][5])
* **EBS volumes** (size and type) cost money. Keep the root volume small (8 GiB) and delete volumes on termination if you don‚Äôt need them.
* **Elastic IPs**: if allocated but not associated you may be charged ‚Äî release them when done.
* **Snapshots** and **data transfers** (egress) can incur costs.
* **CloudWatch detailed monitoring** and reserved public IPs may also add charges.

Always monitor the **Billing & Cost Explorer** and set alerts. ([AWS Documentation][2])

# 5. Clean up when done (avoid surprises) üßπ

To avoid later charges:

* **Stop** the instance if you want to keep the volume (billing for EBS still applies for stopped instances).
* **Terminate** the instance to delete the instance and attached root volume (if configured).
* **Delete EBS volumes** not in use.
* **Release Elastic IPs** that you aren't using.
* **Delete snapshots** you no longer need.
* Check **Billing ‚Üí Free Tier** usage to confirm you‚Äôre inside limits. ([AWS Documentation][2])

Console quick steps: Instances ‚Üí select ‚Üí Actions ‚Üí Instance State ‚Üí Terminate/Stop.
CLI example to terminate:

```bash
aws ec2 terminate-instances --instance-ids i-0123456789abcdef0
```

# 6. Troubleshooting & tips üîß

* **Can‚Äôt SSH?** Check Security Group (port 22), the public IP, and that you‚Äôre using the correct username for the AMI (`ec2-user`, `ubuntu`, `admin`, etc.).
* **Lost key pair?** You can‚Äôt download it again. Use EC2 Instance Connect or create an AMI of the instance and launch a new instance with a new key, or attach the root volume to a helper instance to fix authorized\_keys (advanced).
* **Want a web server?** Install `nginx` or `httpd` after connecting. Open port 80 in the Security Group.
* **Automate cleanup** by adding tags and lifecycle scripts to remove dev/test instances after X hours.
