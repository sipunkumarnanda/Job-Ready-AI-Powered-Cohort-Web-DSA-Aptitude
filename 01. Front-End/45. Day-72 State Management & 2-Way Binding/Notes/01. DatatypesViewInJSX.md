## üß™ What's Happening Here?

You are trying to **render multiple JavaScript datatypes inside JSX** to see how they appear on the browser view.

### ‚úÖ Let's Understand What Renders and Why

| Value       | Will Render? | Reason                                                                                                  |
| ----------- | ------------ | ------------------------------------------------------------------------------------------------------- |
| `Number`    | ‚úÖ Yes        | Numbers are directly converted to string in JSX.                                                        |
| `String`    | ‚úÖ Yes        | Strings display as text.                                                                                |
| `Boolean`   | ‚ùå No         | `true` or `false` will **not render** on screen; React ignores them.                                    |
| `null`      | ‚ùå No         | `null` is **ignored** by JSX.                                                                           |
| `undefined` | ‚ùå No         | `undefined` is also **ignored**, won't appear.                                                          |
| `Array`     | ‚ö†Ô∏è Partial   | React will render elements that it **can**, and **skip** `undefined`, `null`, `true`, `false`.          |
| `Object`    | ‚ö†Ô∏è Partial   | You **cannot render an object** directly ‚Äî only access its properties and render them (e.g. `obj.name`) |

---

## ‚ö†Ô∏è Why JSX Ignores `true`, `false`, `null`, and `undefined`?

JSX is syntactic sugar for `React.createElement(...)`. React **automatically ignores** these values in the rendering process to avoid displaying meaningless data.

### ‚ùå These will NOT render:

```jsx
<h2>{true}</h2>       // Nothing will be shown
<h2>{undefined}</h2>  // Nothing will be shown
<h2>{null}</h2>       // Nothing will be shown
```

### ‚úÖ These are okay:

```jsx
<h2>{10}</h2>         // Displays: 10
<h2>{"Hello"}</h2>    // Displays: Hello
<h2>{obj.name}</h2>   // Displays: Sipun
```

## ‚ö†Ô∏è Rendering an Array: Special Case

```jsx
let arr = [<h1>Heyy</h1>, 10,20,30,40,50, "hello", undefined, true, null, 100]
<h2>Array: {arr}</h2>
```

### Output:

React will try to render each item in the array:

* `<h1>Heyy</h1>` ‚ûú ‚úÖ rendered
* `10,20,30,...` ‚ûú ‚úÖ rendered
* `"hello"` ‚ûú ‚úÖ rendered
* `undefined`, `null`, `true` ‚ûú ‚ùå ignored
* `100` ‚ûú ‚úÖ rendered

So, you‚Äôll **see everything except the ignored types**. But they won't throw an error.

## üîÅ Two-Way Binding (Intro)

Two-way binding = syncing the **view (UI)** and the **state (JS variable)**.

You‚Äôre not using state (`useState`) here, so technically this is **not** two-way binding yet. You are just rendering static variables.

> You'll explore **two-way binding** fully when you use `useState()` and update the UI dynamically from input fields etc.

## üß© Final Summary

* JSX **renders only renderable data types**: numbers, strings, JSX elements.
* JSX **ignores**: `true`, `false`, `null`, `undefined`.
* You cannot directly render an object ‚Äî must access specific properties.
* Arrays render valid content, skip what can't be rendered.
* This example is a good **exploration of how JSX behaves** with various data types.
