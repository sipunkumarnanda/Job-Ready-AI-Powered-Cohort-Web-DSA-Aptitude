## âœ… What is `useCallback`?

**`useCallback` is a React hook** that returns a **memoized version of a function** â€” which means the function will only change **if its dependencies change**.

> Itâ€™s used to **prevent unnecessary re-creation** of functions on every render.

## ðŸ’¡ Why does this matter?

In React, **functions are re-created every time a component re-renders**, just like variables and other code.

Example:

```jsx

  const handleClick = () => {
    console.log("Button clicked");
  };

  return <button onClick={handleClick}>Click me</button>;
};
```

Every time this component re-renders, `handleClick` is **created again** â€” even if the function **looks the same**.

This usually isn't a problem â€” but when:

* You **pass functions as props** to child components
* Or use them in **`useEffect` or `useMemo` dependencies**

â€¦React may behave inefficiently or trigger **unnecessary updates**.

---

## âœ… What `useCallback` does:

```jsx

  // logic
}, [dependencies]);
```

* If `dependencies` don't change â†’ React gives you the **same function**
* If `dependencies` change â†’ React **creates a new function**

This saves memory and prevents unnecessary updates.

---

## ðŸ“¦ When Should You Use `useCallback`?

Use it when:

1. You're passing functions to child components that are **memoized** (`React.memo`)
2. You're using a function **inside `useEffect` or `useMemo` dependencies**
3. The function is **expensive to create** or causes **re-renders**

**Avoid** it for every function â€” use it **only when needed**, because unnecessary `useCallback` can **overcomplicate** your code.

## ðŸ” `useCallback` vs `useMemo`

| Feature  | `useCallback`       | `useMemo`                           |
| -------- | ------------------- | ----------------------------------- |
| Returns  | A **function**      | A **value** (number, array, object) |

| Prevents | Function recreation | Value recalculation                 |
